🔥 3 основных подхода

Разберём по уровню сложности 👇

🟢 Уровень 1 — Threshold tuning (быстрый win)
Идея:

Подобрать оптимальный порог similarity

Как:
Берёшь пары:
same cluster → label = 1
different → label = 0
Считаешь cosine similarity
Строишь ROC / PR curve
Результат:

👉 оптимальный threshold вместо “на глаз”

🟡 Уровень 2 — Supervised similarity model
Идея:

Обучить модель:

“насколько два запроса принадлежат одному кластеру”

Формат данных:
{
  "query1": "...",
  "query2": "...",
  "label": 1
}
Модели:
Sentence-BERT (fine-tuning)
cross-encoder (лучше 👈)
Что меняется:

Вместо:

cosine(q1, q2)

Ты используешь:

model.predict(q1, q2)
🔥 Плюс:
учитывает intent
учитывает формулировку
сильно лучше для SEO
🔴 Уровень 3 — Clustering as ML task
Идея:

Обучить модель сразу делать кластеры

Подходы:
1. Pairwise + Graph
модель → вероятность связи
строишь граф
находишь компоненты
2. Contrastive learning
обучаешь embedding space
где:
похожие → ближе
разные → дальше

👉 Это уже уровень production AI

⚙️ Как встроить в твой пайплайн
keywords
   ↓
embeddings (base)
   ↓
similarity model (fine-tuned)
   ↓
graph clustering
   ↓
SERP validation
💡 Очень сильный гибрид (рекомендую)
1. Embeddings (recall)
быстро находишь кандидатов
2. ML модель (precision)
решает: вместе или нет
3. SERP (ground truth)
финальная проверка
