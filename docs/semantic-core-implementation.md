# Семантическое ядро сайта: что это и как собрать в этом проекте

## Что такое семантическое ядро
Семантическое ядро сайта это структурированный набор поисковых запросов, который:
- отражает реальный спрос аудитории
- разбит на тематические кластеры
- размечен по пользовательским интентам
- привязан к конкретным страницам сайта

Итоговая цель ядра: превратить спрос из поиска в понятный план структуры, контента и внутренних ссылок.

## Как это ложится на текущий проект
В проекте уже есть основные блоки для сборки ядра:
- импорт ключей из файлов и ручное добавление
- хранение ключей и кластеров в БД
- кластеризация и сохранение результатов
- отдельные страницы для работы с ключами и кластерами

Это позволяет собрать ядро не в таблицах вручную, а в управляемом рабочем процессе внутри сервиса.

## Пошаговый процесс реализации

### 1. Сбор полного пула запросов
1. Загрузить ключи через импорт файлов (CSV, Excel, TXT, DOCX).
2. Добавить недостающие ключи вручную в рабочем интерфейсе.
3. Объединить источники: коммерческие, информационные, брендовые, навигационные.

Результат: единый список ключей по домену.

### 2. Очистка и нормализация
1. Удалить дубли и мусорные формулировки.
2. Привести ключи к единому формату (регистр, орфография, шумовые символы).
3. Исключить нерелевантные и слишком общие фразы, не несущие бизнес-ценность.

Результат: чистый пул ключей для кластеризации.

### 3. Кластеризация
1. Запустить кластеризацию по домену.
2. Проверить связность групп: можно ли покрыть запросы внутри кластера одной страницей.
3. При необходимости скорректировать число кластеров и повторить запуск.

Результат: тематические группы запросов, пригодные для структуры сайта.

### 4. Разметка интентов
Для каждого кластера назначить основной интент:
- informational
- commercial
- transactional
- navigational

Если кластер смешанный:
1. Разделить его на подгруппы.
2. Либо зафиксировать primary и secondary интент.

Результат: интентно-ориентированное ядро.

### 5. Привязка к структуре сайта
1. Сопоставить каждый кластер существующей странице или новой целевой странице.
2. Проверить, что один кластер решает одну пользовательскую задачу.
3. Устранить каннибализацию между страницами с похожими интентами.

Результат: карта соответствия кластер -> URL.

### 6. Приоритизация
Оценить каждый кластер по трем осям:
- потенциальный спрос
- бизнес-ценность
- сложность реализации

Сформировать волны внедрения:
1. Быстрые победы.
2. Стратегические кластеры.
3. Long-tail поддержка.

Результат: реалистичный план внедрения ядра.

### 7. Контентные задания на основе кластеров
По каждому кластеру оформить:
- целевой URL
- тема страницы и H1
- обязательные подтемы
- FAQ
- внутренние ссылки на соседние кластеры

Результат: кластер превращен в исполняемое ТЗ для контента.

### 8. Цикл обновления ядра
1. Раз в 2-4 недели догружать новые запросы.
2. Повторно кластеризовать изменившиеся группы.
3. Перепроверять интенты по изменениям выдачи.
4. Обновлять приоритеты по фактическим метрикам.

Результат: ядро остается актуальным и не деградирует со временем.

## Критерии готовности
Семантическое ядро считается готовым, если:
1. Все ключи распределены по понятным кластерам без значимого мусора.
2. У каждого кластера есть интент и целевая страница.
3. Нет заметной каннибализации между URL.
4. Есть контентный и технический план внедрения.
5. Есть регламент регулярного обновления ядра.

## Рекомендуемый рабочий артефакт
Для операционной работы удобно вести таблицу с полями:
- cluster_id
- cluster_name
- intent_primary
- intent_secondary
- keywords_count
- priority
- target_url
- page_type
- status
- owner
- notes

Такое представление позволяет быстро переходить от анализа к публикации и обновлению страниц.
